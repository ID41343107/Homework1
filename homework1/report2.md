# Problem 2: GeneratePowerset

## 解題說明

問題描述：

給定一個大小為 n 的集合 S，要求生成 S 的冪集（Power Set），即 S 的所有可能子集，包括空集。

解題策略：

使用遞迴（Recursion) 來生成冪集。對集合中的每一個元素，都有兩種選擇：

1. 不加入當前元素到子集。

2. 加入當前元素到子集。

遞迴函數 G 透過索引 i 依序處理集合的每個元素，並用變數 C 記錄當前子集的大小。當遞迴到集合的最後一個元素時，將當前子集輸出，即完成一個子集的生成。

步驟流程：

1. 使用者輸入集合大小 n 與元素值。

2. 初始化兩個陣列：

   S[] 儲存原集合元素

    c[] 儲存當前遞迴生成的子集

3. 呼叫遞迴函數 G(S, c, 0, n, 0)：

   i=0：從集合的第一個元素開始處理

   C=0：子集初始大小為 0

4. 遞迴過程中，每個元素有兩種選擇（加入 / 不加入），透過回溯法生成所有子集。

5. 遞迴完成後，輸出所有子集，即為集合的冪集。

## 程式實作
以下為主要程式碼：
```cpp
#include <iostream>
#include <string>
using namespace std;

// S[]:原集合 s[]:當前子集 i:目前處理到S[i] N:集合大小 n:當前子集大小
void G(string S[], string s[], int i, int N, int n) {   // 遞歸生成冪集函數
    if (i == N) {
        cout << "(";
        for (int j = 0; j < n; j++) {
            cout << s[j];
            if (j != n - 1) cout << ","; 
        }
        cout << ")" << endl;
        return;
    }

    // 不加入當前元素 S[i]，繼續處理下一個元素
    G(S, s, i + 1, N, n);
    // 加入當前元素 S[i] 到子集
    s[n] = S[i];
    // 繼續處理下一個元素，子集大小 C 增加 1
    G(S, s, i + 1, N, n + 1);
}

int main() {
    int N;
    cout << "請輸入n個數 : ";
    cin >> N;

    string* S = new string[N];
    string* c = new string[N];

    cout << "請輸入集合元素 : ";
    for (int i = 0; i < N; i++) {
        cin >> S[i];
    }

    cout << "集合的冪集為:" << endl;
    G(S, c, 0, N, 0);

    delete[] S;
    delete[] c;

    return 0;
}

```
## 效能分析

1. 時間複雜度分析 (Time Complexity)

 冪集的生成過程中，每一個元素都有兩種選擇：

      加入子集

      不加入子集

因此，若集合 S 含有 n 個元素，則總共有 2𝑛種子集。

在每個子集生成時，程式需要輸出最多 n 個元素，因此總操作次數約為：𝑂(𝑛×2^𝑛)

推導說明：

對於每個元素，都會遞迴呼叫兩次（加入與不加入）。

遞迴樹共有 2𝑛 個節點（即子集數量）。

每次輸出最多 n 個元素，因此整體時間為 𝑂(𝑛×2^𝑛)

範例：

| n	| 子集數量	| 時間複雜度 |
|----|--------|------------|
| 1	| 2     | O(2)       |
| 2	| 4  	  | O(4)       |
| 3	| 8	  | O(8)       |
| 4	| 16	  | O(16)      |
| n	| 2𝑛    | O(n × 2^n) |

結論：

冪集生成的時間複雜度為 O(n × 2^n)，屬於指數級增長。

2. 空間複雜度分析 (Space Complexity)

空間主要由兩部分構成：

 2.1 遞迴堆疊深度 (Call Stack)

      每次遞迴處理一個元素，最深遞迴深度為 n。

      因此堆疊使用空間為 O(n)。

 2.2 暫存子集陣列 (current subset)

      儲存目前正在生成的子集，最大長度為 n。

      綜合以上，空間複雜度為：O(n)

3. 效能結論

|分析項目	| 結果 |	說明 |
|-----------|--------------|-------------------------------|
| 時間複雜度	| O(n × 2^n)	| 每個元素皆有加入與不加入兩種選擇      |
| 空間複雜度	| O(n)	      | 由遞迴深度與子集暫存空間決定          |
| 適用範圍	| n ≤ 15	      | n 過大時輸出子集數量過多，運算時間長  |

總結：

冪集的生成屬於典型的指數時間遞迴問題。

雖然對小集合運算快速，但當 n 增大時，子集數量急劇增加。

實務上若 n > 20，列舉所有子集會非常耗時與佔記憶體。

## 申論及開發報告
使用遞迴來做GeneratePowerset 的原因如下：

1. 遞迴直觀：

   程式邏輯清楚，對每個元素只考慮加入或不加入兩種情況。

2. 回溯技巧

   加入元素後遞迴返回，子集大小自動恢復，不需額外操作。

3. 動態陣列：
   
   利用 new 配置陣列大小，執行時可依使用者輸入決定集合大小，最後釋放記憶體。

4. 輸出完整：
   
   可以生成包含空集在內的所有子集，順序為遞迴生成順序。

總結

這個程式用「遞迴」的方式，把集合中每個元素分成兩種情況來考慮：「要」或「不要」放進子集，最後就能列出所有可能的組合，也就是冪集。整個程式邏輯很清楚，寫法也簡單，能很好地說明遞迴和回溯的概念。不過，因為每多一個元素，子集數量就會變成兩倍（也就是 2ⁿ 的成長），所以當 n 很大時，程式會變得很慢、也很吃記憶體。
總結來說，這個程式可以完整地產生所有子集，也能幫助理解遞迴是怎麼一層層運作的。
